\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Solution\+Field\+Type$>$\newline
class Neo\+FOAM\+::dsl\+::\+Runge\+Kutta$<$ Solution\+Field\+Type $>$}

Integrates in time, using Sundials, a PDE expression using the Runge-\/\+Kutta method. 


\begin{DoxyTemplParams}{Template Parameters}
{\em Solution\+Field\+Type} & The Solution field type, should be a volume or surface field.\\
\hline
\end{DoxyTemplParams}
Implements explicit Runge-\/\+Kutta time integration using the Sundials library. The class manages Sundials vectors and memory through RAII principles, handling the conversion between internal field representations and Sundials\textquotesingle{} N\+\_\+\+Vector format. Supports various (at present explicit) Runge-\/\+Kutta methods which can be specified through the dictionary configuration. The main interface for a solve is through the {\ttfamily solve} function.

\begin{DoxyNote}{Note}
Useful Sundials documentation below, currently we have implemented only an explicit Runge-\/\+Kutta interface, this simplifies things considerably as compared to some of the examples\+: Initialization (and order thereof)\+: \href{https://sundials.readthedocs.io/en/latest/arkode/Usage/Skeleton.html}{\texttt{ https\+://sundials.\+readthedocs.\+io/en/latest/arkode/\+Usage/\+Skeleton.\+html}} Sundials-\/\+Kokkos\+: \href{https://sundials.readthedocs.io/en/latest/nvectors/NVector_links.html\#the-nvector-kokkos-module}{\texttt{ https\+://sundials.\+readthedocs.\+io/en/latest/nvectors/\+NVector\+\_\+links.\+html\#the-\/nvector-\/kokkos-\/module}} Sundials Contexts (scroll to bottom eg, they don\textquotesingle{}t like copying)\+: \href{https://sundials.readthedocs.io/en/latest/sundials/SUNContext_link.html\#c.SUNContext_Create}{\texttt{ https\+://sundials.\+readthedocs.\+io/en/latest/sundials/\+SUNContext\+\_\+link.\+html\#c.\+SUNContext\+\_\+\+Create}}
\end{DoxyNote}
\begin{DoxyWarning}{Warning}
For developers\+:
\begin{DoxyEnumerate}
\item This class uses Sundials-\/\+Kokkos vectors for computation, which are immediately wrapped as Sundials N\+\_\+\+Vectors. After initialization, only interact with the N\+\_\+\+Vector interface as per Sundials guidelines.
\item The Sundials context is supposed to only be created and freed once in a program, making copying less desirable, see above. However we need to copy, so the context is placed in a shared\+\_\+ptr to prevent early freeing. Please read the documentation about multiple, concurrent solves (you must not do them). You have been warned. 
\end{DoxyEnumerate}
\end{DoxyWarning}
